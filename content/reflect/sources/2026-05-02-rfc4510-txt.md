RFC 4510 (Standards Track, June 2006) is the umbrella document for the LDAPv3 Technical Specification. Edited by [[reflect/entities/kurt-zeilenga|Kurt Zeilenga]] of the [[reflect/entities/openldap-foundation|OpenLDAP Foundation]] and produced by the IETF LDAPBIS Working Group, it defines which documents collectively constitute "LDAP" or "LDAPv3" and provides normative references to each companion RFC. It supersedes RFC 3377 — the previous LDAPv3 road map — along with the entire 1997–2001 specification series.

## The LDAPv3 Technical Specification

The RFC 4510 series consists of nine documents. RFC 4510 is the road map; the normative technical content lives in the companion RFCs:

| RFC | Title |
|---|---|
| [[reflect/sources/2026-05-02-rfc4511-txt\|RFC 4511]] | LDAP: The Protocol |
| RFC 4512 | LDAP: Directory Information Models |
| [[reflect/sources/2026-05-02-rfc4513-txt\|RFC 4513]] | LDAP: Authentication Methods and Security Mechanisms |
| RFC 4514 | LDAP: String Representation of Distinguished Names |
| [[reflect/sources/2026-05-02-rfc4515-txt\|RFC 4515]] | LDAP: String Representation of Search Filters |
| RFC 4516 | LDAP: Uniform Resource Locator |
| [[reflect/sources/2026-05-02-rfc4517-txt\|RFC 4517]] | LDAP: Syntaxes and Matching Rules |
| RFC 4518 | LDAP: Internationalized String Preparation |
| RFC 4519 | LDAP: Schema for User Applications |

The terms "LDAP" and "LDAPv3" informally denote the protocol as a whole. Documents that formally identify this specification should reference RFC 4510 normatively rather than citing individual companion RFCs.

## Relationship to X.500

RFC 4510 defines [[reflect/concepts/ldap|LDAP]] as an X.500 access mechanism. LDAP servers MUST act in accordance with the X.500 (1993) ITU-T data and service models — specifically the Directory Models (X.501) and Abstract Service Definition (X.511) — but are not required to use X.500 protocols internally. LDAP may be implemented over any underlying directory system as long as the X.500 data and service models are not violated at the LDAP interface.

The RFC explicitly incorporates only X.500 (1993). Later X.500 revisions do not automatically apply to this technical specification.

## Relationship to Obsolete Specifications

RFC 4510 obsoletes the prior LDAPv3 technical specification, which consisted of RFC 3377 and the RFCs it enumerated. The 2006 reorganization significantly redistributed concerns across new single-subject documents:

| Obsoleted | Primary replacement |
|---|---|
| RFC 2251 (LDAPv3 core protocol) | [[reflect/sources/2026-05-02-rfc4511-txt\|RFC 4511]] (protocol), RFC 4512 (information models), [[reflect/sources/2026-05-02-rfc4513-txt\|RFC 4513]] (auth) |
| RFC 2252 (Attribute Syntaxes) | RFC 4512 + [[reflect/sources/2026-05-02-rfc4517-txt\|RFC 4517]] |
| RFC 2253 (DN String Representation) | RFC 4514 |
| RFC 2254 (Search Filter Syntax) | [[reflect/sources/2026-05-02-rfc4515-txt\|RFC 4515]] |
| RFC 2255 (LDAP URL Format) | RFC 4516 |
| RFC 2256 (X.500 Schema for User Apps) | RFC 4519 + RFC 4512 + [[reflect/sources/2026-05-02-rfc4517-txt\|RFC 4517]] |
| RFC 2829 (Authentication Methods) | [[reflect/sources/2026-05-02-rfc4513-txt\|RFC 4513]] |
| RFC 2830 (StartTLS extension) | [[reflect/sources/2026-05-02-rfc4513-txt\|RFC 4513]] + [[reflect/sources/2026-05-02-rfc4511-txt\|RFC 4511]] |
| RFC 3771 (IntermediateResponse) | [[reflect/sources/2026-05-02-rfc4511-txt\|RFC 4511]] |
| RFC 3377 (previous road map) | RFC 4510 (this document) |

RFC 4518 (Internationalized String Preparation) is new to this revision — no predecessor existed in the 1997 series.

## Extensibility

LDAP is explicitly described as an extensible protocol. Extensions defined in other documents MUST be genuinely optional per BCP 118 (RFC 4521, "Considerations for LDAP Extensions"). IANA registration for LDAP schema elements, controls, extended operations, and features is governed by BCP 64 (RFC 4520, "IANA Considerations for LDAP") — both BCP documents authored by [[reflect/entities/kurt-zeilenga|Zeilenga]].
