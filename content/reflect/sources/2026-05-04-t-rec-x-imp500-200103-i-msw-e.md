The **Directory Implementors' Guide Version 14** (COM 7-45-E, March 2001) is the immediate predecessor to [[reflect/sources/2026-05-04-t-rec-x-imp500-200109-i-msw-e|Version 15]] and the **final version of the Guide to document Technical Corrigenda for the 2nd edition** (1993/1995) of the X.500-series recommendations. Agreed at the 2 February 2001 meeting of [[reflect/entities/itu-t|ITU-T]] Study Group 7, it was produced jointly by ITU-T Study Group 7 and [[reflect/entities/iso-iec-jtc1-sc6|ISO/IEC JTC 1/SC 6]], edited by [[reflect/entities/hoyt-kesterson|Hoyt L. Kesterson II]].

The Guide's own note is explicit: *"This version will be the last Implementors Guide for the 2nd edition. Readers still using the 2nd edition are advised to keep this document."* Version 15 (August 2001) removed the 2nd edition appendix entirely; V14 is therefore the authoritative reference for how the 2nd (1993) edition was corrected before implementers were expected to migrate to the 3rd or 4th edition.

## Structure

Version 14 is organized into three content appendices and three administrative appendices:

- **Appendix A**: Technical Corrigenda to the **2nd edition** (1993/1995) — covers X.501, X.511, X.518, X.519, X.520, X.509, X.525 (TC1–4 per part as applicable)
- **Appendix B**: Technical Corrigenda to the **3rd edition** (1997/1998) — all parts including X.530; covers DRs up through DR 269
- **Appendix C**: Technical Corrigenda to the **4th edition** (2000/2001) — notes that all currently approved TCs were incorporated into the published edition
- **Appendix D**: Summary of all defect reports (DR 075 onward), with status and disposition per edition
- **Appendix E**: Defect report form
- **Appendix F**: Defect Resolution Committee member roster

## Key 2nd Edition Corrigenda

### X.509 TC1 (1993 Edition): Introducing v3 Certificates (DR 128)

The most historically significant correction in V14 is X.509 Technical Corrigendum 1, which **introduced the v3 certificate format into the 2nd edition specification**. Prior to this TC, the 1993 X.509 text supported only v1 and v2 certificates. TC1 added:

- A new `v3(2)` value in the `Version` production
- An `extensions [3] Extensions OPTIONAL` field at the end of the `TBSCertificate` SEQUENCE
- The `Extensions ::= SEQUENCE OF Extension` type
- The `Extension ::= SEQUENCE { extnId, critical BOOLEAN DEFAULT FALSE, extnValue OCTET STRING }` structure
- The `EXTENSION` information object class with `&id` and `&ExtnType` fields
- CRL versioning: a `version OPTIONAL` field in `CertificateList`, plus `crlEntryExtensions` and `crlExtensions` components
- Notes defining the semantics of critical and non-critical extensions in CRLs

The v3 certificate format that is universal in TLS/HTTPS today entered the normative 2nd-edition specification through a Technical Corrigendum. The 3rd edition (1997) incorporated these corrections as baseline. See [[reflect/concepts/x509-pki|X.509 PKI]] for the full extension and certification path semantics.

### Y2K Corrections (Defect Report 211)

DR 211 was applied across all active editions and all affected parts. In V14:

- **2nd edition**: TC4 for X.501, X.511, X.518, X.520 — replacing `UTCTime` with a `Time ::= CHOICE { utcTime UTCTime, generalizedTime GeneralizedTime }` type, with two-digit year rationalization rules (00–49 → 2000+, 50–99 → 1900+) and a prohibition on UTCTime for dates beyond 2049
- **3rd edition**: TC1 or TC2 for the same parts — the same correction applied to 3rd-edition text
- **4th edition**: incorporated into the published text

The same rationalization rules apply across X.501 operational binding validity periods, X.511 security parameters, X.518 chaining arguments, and X.520 time-based matching rules.

### X.509 TC3 (2nd Edition): Signature Semantics (DRs 080, 092, 100, 177, 183, 194)

TC3 revised clause 9's signature ASN.1, replacing the ambiguous `HASHED` parameterized type with a new `ENCRYPTED-HASH` definition and rewriting `SIGNATURE` to use it:

```asn1
ENCRYPTED-HASH { ToBeSigned } ::= BIT STRING ( CONSTRAINED BY {
    -- must be the result of applying a hashing procedure to the BER-encoded octets
    -- of a value of -- ToBeSigned -- and then applying an encipherment procedure ... })
SIGNATURE { ToBeSigned } ::= SEQUENCE {
    algorithmIdentifier  AlgorithmIdentifier,
    encrypted            ENCRYPTED-HASH { ToBeSigned }}
```

DR 100 added rules for signing data that contains unknown protocol extensions — establishing the "preserve the encoding" principle: a DSA that must subsequently re-sign received data it does not fully understand shall preserve its original encoding rather than re-encoding it. DR 177 added canonical encoding rules for UTCTime and GeneralizedTime to the list of signing requirements. DR 183 added `encipherOnly (7)` and `decipherOnly (8)` bits to the `keyUsage` BIT STRING.

### X.509 TC2 (2nd Edition): Terminology Corrections (DRs 077, 078, 083, 084)

These corrections replaced the term "secret key" with "private key" throughout X.509 and clarified that user certificates are formed "by off-line means" rather than the prior ambiguous phrasing. DR 084 added guidance on bit ordering in padding.

### X.501 TCs (2nd Edition): Schema and DIT Structure

- **TC1** corrected the `STRUCTURE-RULE` ASN.1 definition, the `superiorStructureRules` component semantics, and the `ModifyOperationalBindingArgument` newAgreement component.
- **TC2** fixed the `Attribute` type to allow zero values (`SET SIZE (0..MAX)`), clarified subschema administrative area boundaries, and added an X.660 reference for OSI registration procedures.
- **TC3** fixed the `superiorKnowledge` operational attribute to be multi-valued (`SET OF AccessPoint`), corrected First Level DSA definitions, and clarified the `supr` DSE type for superior reference storage.
- **TC4** applied the DR 211 `UTCTime → Time` correction to operational binding validity.

### X.518 Distributed Operations TCs (2nd Edition)

TCs 1–4 corrected the distributed operation procedures: ChainedArguments field ordering, Find DSE procedure steps, search procedure corrections, and loop avoidance. Notably, TC2 added a `referralRequests` data structure and a loop-avoidance check (DR 119) that prevents infinite chaining loops when following referrals — a critical safety property for distributed operation. See [[reflect/concepts/x500-distributed-directory|X.500 Distributed Directory]] for the architectural context.

### X.525 Replication TCs (2nd Edition)

TC1 added `subordinates BOOLEAN DEFAULT FALSE` to the `UnitOfReplication` type (DR 182), enabling shadow consumers to request that subordinate entries — not just knowledge references — be copied. TC3 added `subordinates` to the `UnitOfReplication` ASN.1 alongside access control information handling. See [[reflect/concepts/x500-directory-replication|X.500 Directory Replication]] for context.

## 3rd Edition Corrigenda (Subset of V15)

The 3rd edition TCs in V14 represent the state of ballot proceedings as of February 2001. V15 (August 2001) added further DRs resolved after V14's cutoff. Key 3rd edition TCs documented in V14:

- **X.501 TC1**: Superior reference fixes (DRs 173, 179, 189, 205); `superiorKnowledge` multi-valued; first-level DSA corrections
- **X.501 TC2**: DIRQOP/GULS deprecation (DRs 228, 229, 230); Enhanced Security ASN.1 module rewrite; `AttributeIntegrityInfo` corrections; `CommonResultsSeq` added
- **X.511 TC1/TC2**: Alias dereferencing flowcharts, security parameter semantics, filter three-valued logic (DR 224), SimpleCredentials ASN.1 (DR 231), joint-iso-ccitt → joint-iso-itu-t rename (DR 232)
- **X.518 TC1/TC2**: ModifyDN/shadow interaction, search/list procedure corrections, `AccessPointInformation` ASN.1 (DR 235), request decomposition constraint (DR 234), DIRQOP removal
- **X.519 TC1/TC2**: Conformance text, GULS/SESE application context removal (DR 228), ASN.1 module corrections (DR 236)
- **X.520 TC1/TC2**: UTCTime rationalization (DR 211), attribute type corrections (DRs 237, 238, 241, 270)
- **X.521 TC1**: Missing imports fix (DR 239)
- **X.509 TC1/TC2**: Certification path processing rewrite (DR 222), certificate policy architecture, `inhibitAnyPolicy` extension, extension criticality semantics (DR 244), CRL matching rules (DRs 226, 227, 240), cross-certificate renaming (DRs 257, 258)
- **X.525 TC1/TC2**: Subordinates shadowing, access control in shadows (DRs 182, 186, 187, 208, 243)
- **X.530 TC1**: ASN.1 module corrections (DR 252)

## 4th Edition Status

The 4th edition summary in V14 notes that all approved TCs for the 4th edition (X.501, X.511, X.518, X.519, X.520, X.509) were incorporated into the published edition text. V14 lists these DTCs as "Incorporated into published edition" — confirming that the 4th edition launched with its corrigenda already merged.
