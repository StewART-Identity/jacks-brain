Don Norman is a cognitive scientist, UX researcher, and author of *The Design of Everyday Things* (1988, revised 2013). He co-founded the Nielsen Norman Group and served as VP of Advanced Technology at Apple.

## Key Concepts

Norman's most influential contributions to interface design include:

- **Affordances** — the perceived properties of an object that suggest how it should be used
- **Feedback** — the system's immediate, legible response to a user action
- **Conceptual models** — the user's mental model of how a system works; good design makes this model accurate
- **Gulf of evaluation / Gulf of execution** — the gaps between what a system does and what a user expects, in both intent and interpretation

## "The System Is Fighting Me"

[[collection/entities/jack-stewart|Jack Stewart]] invokes Norman's framing in Rule 6 of [[collection/sources/2026-04-25-jacks-rules-for-website-design|Jack's Rules for Website Design]]: when a form re-asks the user for information they have already provided, the system communicates that it does not remember their prior effort. Norman would describe this as a conceptual model failure — the user's model (the system knows my data) diverges from the system's behavior (re-enter your data), producing the feeling of adversarial resistance.

Every redundant re-entry is surplus cognitive work the system chose not to absorb. See [[collection/concepts/cognitive-load|cognitive load]] for the theoretical underpinning.
