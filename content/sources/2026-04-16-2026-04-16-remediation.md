## Key Findings

### Highest-Risk Item: Windows Server 2022 Cumulative Update
Risk score 4,028,451 — the largest single remediation. Affects 35 Windows hosts including all `DCPD-*` domain controllers, `AZC-IDPR-*`, `ADF-IDPR-*`, and LDAP replication servers (`rlad-idpr-*`). Patch: KB5071547.

### Unfixable Vulnerabilities (#3)
44 assets carry vulnerabilities with **no public solution**. Recommended responses: hardening, access restriction, software removal, or decommission. This is the third-highest risk item (1,348,461), indicating meaningful exposure that cannot be patched away.

### Heavy Red Hat / Linux Footprint
Items 2, 8–11, 15–23 are all Red Hat package upgrades. The same ~43 Linux servers appear repeatedly across kernel, python3, bind, sssd, and sssd-related libraries. These servers include `ams-idpr-*`, `duo-idpr-*`, `iam-*`, `idm*`, `rad-idpr-*`, `rllx-*`, `sso*.ardc.*`, and `sso*.gabdcn.*` hosts.

### CVE-2022-0001 — Branch History Injection (Intel BHI)
Item 25 affects 47 Windows assets — the broadest footprint of any single CVE. Mitigation requires registry key changes (`FeatureSettingsOverride`/`FeatureSettingsOverrideMask`) on all Intel-based Windows domain controllers and IAM servers. This is a microarchitectural side-channel vulnerability with no complete hardware fix.

### 3DES Cipher Suite Still Enabled (Item 21)
24 assets — primarily [[entities/active-directory|Active Directory]] domain controllers — still support the deprecated 3DES cipher suite. This is a cryptographic weakness on public-facing domain controllers.

### Certificate Padding Check Missing (CVE-2013-3900)
35 Windows assets lack the certificate padding validation fix from 2013. A very old vulnerability still present across the AD infrastructure.

### SSO Servers Accumulate Multiple Issues
`sso1.gabdcn.unt.edu`, `sso1.ardc.unt.edu`, `sso2-test.ardc.unt.edu`, and `sso1-test.gabdcn.unt.edu` appear in items 2, 3, 9, 11, 14–20, 22–23 — Apache Tomcat, kernel, python3, bind, and sssd upgrades are all needed. The SSO layer has the broadest software vulnerability surface.

## Server Infrastructure Naming Patterns Visible

The report reveals the IAM environment naming convention:

- `[service]-[env]-[role][nn].[domain]`
- Environments: `idpr` (production), `iddv` (dev), `ardc` / `gab` (data center sites)
- Services: `ams` (account management), `duo`, `iam`, `idm`/`idmd`/`idmp` (identity management), `rad` (RADIUS), `rllx` (Linux), `rlad` (AD replication), `sso`, `wap` (web app proxy)
- Domains: `untsystem.edu` (Linux/system servers), `ad.unt.edu` / `*.ad.unt.edu` (Windows/AD servers)
