Object model, refs, internals, and history-rewriting safely. The goal of this topic is to understand Git well enough that commands stop feeling like incantations and start feeling like obvious consequences of how the system actually works.

## Foundational pages

- [[upskill/git/object-model|Object model]] — blobs, trees, commits, and tags. The content-addressed database underneath every Git command.
- [[upskill/git/refs|Refs]] — branches, tags, HEAD, and the reflog. Names that point at object IDs.
- [[upskill/git/remotes|Remotes]] — tracking branches, fetch vs. pull vs. push, and how the local repo mirrors a remote.
