# Jack's Brain — Patch (May 19, 2026)

A vocabulary and structure refresh that splits the old "Learn" menu
into three top-level sections (Collect, Search, Reflect) and adds a
new Application section. This is the largest structural rename since
the April 24 patch.

## What's in this patch

### Top-level menu refactor

The old "Learn" menu collapsed three different concerns into one
heading: pipeline pages (Selection, Acquisition, Retention), search
tools (Wiki, Web), and category indexes (Sources, Entities,
Concepts, Synthesis). Three concerns, one menu — every time the user
needed to do something, they had to mentally classify the action
before they could find the menu item.

The new layout splits them apart:

- **Collect** — the cataloging pipeline. What goes in, what's in
  progress, what's been retained.
  - Selection (upload form)
  - Acquisition (live processing status)
  - Retention (chronological audit log, with inline title editing)
- **Search** — query tools.
  - Wiki (in-collection search)
  - Web (external research)
- **Reflect** — the cataloged collection itself.
  - Sources (the per-document summary pages)
  - Entities (people, organizations, tools, systems)
  - Concepts (ideas, theories, frameworks)
  - Synthesis (cross-cutting analysis)
- **Visualize** — graph view of the wiki.
  - Graph (full-page d3 force layout)
  - Help (graph view shortcuts and tips)
- **Application** — meta-pages about the wiki application itself.
  - Help (using the application)
  - Nuke It From Orbit (the wipe-and-reset action, gated behind a
    confirmation dialog)

The directory layout in `content/` mirrors the menu: `content/collect/`,
`content/search/`, `content/reflect/`, `content/visualize/`,
`content/application/`.

### Path renames

Every page in the wiki moved. Wikilinks are auto-resolving in Quartz, so
existing `[[reflect/concepts/oauth]]`-style links continued to work, but
the canonical paths are now:

| Old path                              | New path                              |
|---------------------------------------|---------------------------------------|
| `content/learn/selection.md`          | `content/collect/selection.md`        |
| `content/learn/acquisition.md`        | `content/collect/acquisition.md`      |
| `content/learn/retention.md`          | `content/collect/retention.md`        |
| `content/learn/research.md`           | `content/search/web.md` (renamed)     |
| `content/learn/knowledge.md`          | `content/search/wiki.md` (renamed)    |
| `content/collection/sources/`         | `content/reflect/sources/`            |
| `content/collection/entities/`        | `content/reflect/entities/`           |
| `content/collection/concepts/`        | `content/reflect/concepts/`           |
| `content/collection/synthesis/`       | `content/reflect/synthesis/`          |

Backend code references updated to match: `functions/api/retention.ts`,
`functions/api/source.ts`, `functions/api/originals.ts`, `nuke.ts`'s
`WIPE_DIRS` and `RESET_TEMPLATES`, the catalog workflow's prompts, and
`scripts/catalog.mjs`.

### Application section

The "Nuke It From Orbit" action — the wipe-and-reset endpoint that
restores the wiki to its empty seed state — moved out of the Visualize
menu (where it never belonged) into a new Application section. Sitting
next to a new Application > Help page that documents how the wiki
itself works (vs. Collect/Search/Reflect/Visualize, which document the
collected knowledge).

### Component renames

No component renames in this patch. The Acquisition component name and
its inline/SCSS partners (renamed in the April 24 patch) carry over
unchanged.

## Cleanup steps NOT covered by the rename

A few things to check after deploy:

1. **Cloudflare deployment queue.** A rename of this size produces ~30
   individual commits as the moves land. Cloudflare Pages serially
   rebuilds Quartz for each one (no incremental cache between
   deployments), so the queue takes a few minutes to drain. Watch the
   Workers & Pages dashboard if you're expecting the live site to
   reflect the latest change immediately.

2. **The `deploy.yml` workflow is failing.** It's an orphaned GitHub
   Pages publisher from before the Cloudflare Pages migration and has
   nothing to do with the live deployment. Safe to delete from
   `.github/workflows/` whenever convenient.

3. **MCP server tool descriptions reference the old `content/collection/`
   paths.** The MCP server at `jacks-brain-mcp` ships tool descriptions
   inline in `src/mcp.js`, and several of them still cite
   `content/collection/sources/` as an example path. Functionally
   harmless — the server validates `content/`-prefixed paths
   generically — but worth tidying when a redeploy happens.

## Testing checklist

In a fresh InPrivate window (so Access auth is fresh):

- [ ] Sidebar shows the new five-section layout (Collect, Search,
      Reflect, Visualize, Application) — no leftover "Learn" or
      "Collection" headers.
- [ ] `/collect/acquisition` loads with the Document Processing
      tracker, same as before the rename.
- [ ] `/collect/retention` loads the audit table; inline title edit
      still works.
- [ ] `/reflect/sources` loads the SourcesList table.
- [ ] `/reflect/concepts/ldap` (or any existing concept page) loads at
      its new path. Old paths (`/collection/concepts/ldap`) should
      redirect or return a clean 404 — they should NOT 500.
- [ ] `/search/wiki` loads the search-the-wiki page.
- [ ] `/search/web` loads the external-research page.
- [ ] `/visualize/graph` loads the full graph view, layout dropdown
      still works.
- [ ] `/application/help` and `/application/nuke` load (the Application
      menu's two pages).
- [ ] Acquiring a new document from `/collect/selection` still produces
      a catalog run with source pages at the new
      `content/reflect/sources/` paths.

If any of those fail, check the per-page frontmatter for stale
`updated:` dates or wikilinks pointing at old paths.
